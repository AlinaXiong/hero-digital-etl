# 合同数据迁移\-二级分类映射规则

# **一、****原系统合同类型参考**

|单项合同|收入合同（H\-DS\)|我方收款|
|---|---|---|
||支出合同（H\-DF）|我方付款（最常见的合同类型）|
||收支合同（H\-DJ）|收付款均涉及，比如票务合作|
|框架合同|收入框架（H\-KS）|我方为收入方的框架合同，被收入订单关联|
||支出框架（H\-KF）|我方为支出方的框架合同，被支出订单关联|
||收支框架（H\-KJ）|我方同时涉及到收入及支出的框架合同，可被两种订单关联|
|订单合同|收入订单（H\-OS）|我方为收入方的订单合同，须关联收入框架|
||支出订单（H\-OF）|我方为支出方的订单合同，须关联支出框架|
|合作协议|一般合作协议（H\-P）|不涉及到收入支出的普通合同，如保密协议、反商业贿赂协议等|
||战略合作协议（H\-PZ）|涉及到关键/大型合作的指导性协议，如战略合作协议、合作备忘录等|
|其他|内部合同（H\-N）|公司不同部门或子公司之间签订的合同，该流程提交之前需与财务BP和法务BP确认后方可发起|
||其他合同（H\-Q）|不在上述合同类型范围内的合同类型，基本不涉及|
